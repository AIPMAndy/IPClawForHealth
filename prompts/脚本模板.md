# 📝 口播脚本模板库

> 多种场景的脚本框架，拿来即用

## 第一次用时建议这样做

不要同时追求“专业、爆款、完整、华丽”。第一次先做到这 3 点：

1. 开头 3 秒讲清痛点或反常识
2. 中间只讲 1-3 个关键点
3. 结尾只留 1 个动作

---

## 🎬 通用模板：万能开头

### 模板A：痛点开头

```
[画面：表情严肃]
"是不是总有这种感觉：{痛点描述}？"

[画面：点头]
"我太理解了，因为我{亲身经历/带过学员}..."

[画面：微笑]
"今天我来告诉你{解决方案}，建议收藏"
```

### 模板B：颠覆开头

```
[画面：好奇]
"我发现一个特别有意思的现象：{反常识观点}"

[画面：停顿]
"很多人{大众认知}，但其实{真相}"

[画面：认真]
"今天我用{年数}年的经验，告诉你真相"
```

### 模板C：故事开头

```
[画面：回忆]
"前几天有个{学员/用户}找到我，说{问题}"

[画面：描述]
"他{状态}，已经{尝试过什么}"

[画面：转折]
"后来我教他{方法}，你猜怎么着？{结果}"
```

## 📌 场景1：知识科普

### 结构：是什么 - 为什么 - 怎么办

```
[开头 - 引发兴趣]
"很多人{错误认知}，今天我来告诉你{真相}"

[主体 - 知识讲解]
"其实{原因}，我举个例子{案例}"

{关键点1}...
{关键点2}...
{关键点3}...

[结尾 - 行动]
"如果你{想解决}，记住{核心}，
转发给需要的朋友，我们下期见"
```

### 示例：减肥必须戒碳水？

```
[开头]
"减肥必须戒碳水？这句话害了多少人！

[主体]
我带过上千个学员，发现一个规律：
凡是彻底戒碳水的，最后都反弹了。
为什么？因为{科学原理}...

记住这个顺序：
{具体方法}

[结尾]
减脂不是虐待自己，而是科学地爱自己。
关注我，下期讲{下期预告}"
```

### 再给你一个更完整的健康行业示例

```text
很多人一减脂就不敢点外卖，其实问题不是外卖本身，而是你不会选。

如果你是上班族，先记住三个替换动作：
第一，把奶茶换成无糖茶或黑咖啡。
第二，把炸鸡饭换成鸡胸肉、牛肉或豆腐这类更稳的蛋白质。
第三，把“全都吃完”换成“先吃蛋白质和蔬菜，再决定主食吃多少”。

减脂不是靠忍，而是靠更容易坚持的选择。
如果你想要一份上班族减脂点单清单，评论区扣“外卖”。
```

## 📌 场景2：避坑指南

### 结构：问题 - 原因 - 避坑 - 方法

```
[开头 - 警示]
"这{个数}个{领域}的坑，{踩过的人}都后悔了"

[主体 - 逐个讲解]
"第一个坑：{坑名}
很多人{做法}，结果{后果}
正确的是：{方法}

第二个坑：{坑名}..."

[结尾 - 总结]
"记住{核心}，{好处}，
如果觉得有用，点个赞再走"
```

## 📌 场景4：案例展示

### 结构：背景 - 问题 - 方案 - 结果

```
[开头 - 悬念]
"我前两天刚{完成一件}，{效果}，分享给你"

[背景]
"{用户}找到我的时候，{状态}"

[问题]
"他{问题}，{尝试过什么}，但{没效果}"

[方案]
"我给他{方案}，分{步骤}"

[结果]
"现在{时间}过去了，他{变化}"

[结尾]
"如果你也想{好处}，评论区扣{暗号}"
```

## ⏱️ 时长控制

| 时长 | 结构 | 字数 |
|------|------|------|
| 30秒 | 痛点 + 1个核心点 | ~80字 |
| 60秒 | 痛点 + 核心点 + 结尾 | ~150字 |
| 90秒 | 痛点 + 2-3点 + 案例 | ~220字 |
| 3分钟 | 完整结构 | ~450字 |

## 🎯 脚本检查清单

- [ ] 开头3秒抓住注意力
- [ ] 每15秒有一个小高潮
- [ ] 核心观点重复2次以上
- [ ] 结尾有明确行动指引
- [ ] 口语化、自然流畅

## 🔧 AI 辅助脚本生成

使用以下 prompt 生成定制脚本：

```
你是一位{领域}专家，擅长写口播脚本。
请根据以下信息生成一段{时长}的口播脚本：

主题：{主题}
角度：{角度}
痛点：{痛点}
目标用户：{用户}
时长：{时长}

要求：
1. 口语化、自然
2. 开头3秒抓住注意力
3. 中间有干货、有案例
4. 结尾有行动指引
5. 字数控制在{字数}
```

## ⚠️ 健康行业表达提醒

- 少说“保证有效”“人人适用”“彻底解决”
- 多说“更适合谁”“通常适合什么场景”“如有明显不适建议咨询专业人士”
- 如果讲的是个体案例，要避免包装成普遍结果
- 如果结尾要成交，优先给清单、评估表、训练框架，不要一上来就做强承诺

发布前建议对照 `docs/健康内容合规清单.md` 自查一遍。
